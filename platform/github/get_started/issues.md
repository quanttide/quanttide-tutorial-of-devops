# Issues

Issues 是 GitHub 的任务追踪工具，用于管理功能需求、Bug 报告和改进建议。

## 创建 Issue

1. 进入仓库，点击 **Issues** → **New issue**
2. 填写标题和描述
3. 可选：分配处理人（Assignees）、添加标签（Labels）、关联里程碑（Milestone）
4. 点击 **Submit new issue**

## 基本操作

- **Assignees** — 指定谁负责
- **Labels** — 分类标记（bug、enhancement、question 等）
- **Milestone** — 将 Issue 关联到某个版本目标
- **Close** — 完成后关闭 Issue
